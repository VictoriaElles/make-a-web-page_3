Lesson 1.1: The basic of Web and HTML

How does the Internet work?

There are three components as to how an individual interacts with the internet:
  1. The computer/Browser: A program that allows your computer to display files found on the web
  2. The internet: The world's largest network (often described as a cloud)
  3. Servers: Computers that host the files that make up the web

All of these are interconnected by HTML (Hypertext Transfer Protocol). This video explains this well.

Each file, or web page is a text document written in HTML, HyperText Markup Language. Browsers will in turn interpret HTML. This is made up of:
  • Text content
  • Mark up
  • Reference to other documents, eg. videos, images,...
  • Links

Inline vs. Block

An inline tag groups content but not in box. Examples of such tags are:
  • b: bold
  • em: emphasis (italics)
  • br: break
  • strong
  • image (img)
  • span
A block not only groups content, but also makes an invisible box around it, examples are:
  • p: paragraph
  • form
  • div
Lets take a closer look at "span" and "div" elements.These help to give more structure and create richer texts, but are different. By this I mean, that a span element is in-line and usually used for a small chunk of HTML inside a line (such as inside a paragraph) whereas a div (division) element is block-line (which is basically equivalent to having a line-break before and after it) and used to group larger chunks of code.

Lesson 1.2.: Creating a structured document

Languages of the web

HTML, CSS and JavaScript are the three languages of web developers and the web. Each have a different purpose.

  • HTML controls the "structure" of a web a page. Note that they have a tree-like structure. When programmers talk about the DOM (Document Object Model) they are referring to the tree-lie structure of a web page. HTML is governed by syntax and rules.
  • CSS (Cascading Style Sheets) controls the "style of a web page.

The relationship between indented HTML and boxes

When you read an HTML document as text, you see a what looks like "staircase" of from left to right. The more indented an element is, the more deeply nested its corresponding "box" is. Related not of importance a webpage is made up of numerous boxes.

Text Editors (for programming)

When writing code, programmers use special text editors (like Sublime Text, Scratchpad or CodePen). These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.

The Inheritance principle

But how does CSS work? When coding CSS you start with a selector, for example h3. After the selector there is a left curly brace { and at the bottom there is a matching right curly brace }. Line with this, the Line of code that says that says background-color:black; is a declaration. "Background-color" is a property and "black" is the value of that property.

How to include CSS in your webpage

There are three different methods to include CSS in your webpage.
  • Method 1: write CSS in the head section of your HTML
  • Method 2: Link your HTML to a separate CSS file by creating two separate documents, one for HTML and the other for CSS. These are then linked together by using a link tag (note that this is a void tag).
  • Method 3: Write your style inline with your HTML this is more complex and little confusing. However it is often used so important keep in mind.

The Box Model

As mentioned above, each HTML element on a web page is made up of four boxes or components. Each box is made up of the following (see figure 1):
  • Margin
  • Border
  • Padding
  • Content

As there are so many components to each box, it can be often difficult to get the size of box as you want it. There are two techniques to deal with this:
  1. Set sizes in terms of percentage rather than pixels.
  2. Set the box-sizing attribute to border-box for ever element.
Note that browsers work differently, this may cause one browser to display something else than coded.

Box positioning

As said in lesson 1.1 divs are block elements (as opposed to inline elements), so by default they take up the entire width of a webpage. This as such can make things difficult if you want to reposition a "box".

Thankfully, by adding the rule in CSS element "display:flex;", the CSS will override the static box behaivour and let the divs appear next to each other. This is called the FlexBox layout.

Thinking like a programmer

A key part of this course is to learn to think like a programmer. Five guiding principles for this are:
  1. Procedural thinking: involves creating perfectly clear and unambiguous instructions for a computer to follow.
  2. Abstract thinking: finding similarity, or as programmers would say, generality amongst seemingly different things.
  3. Systematic debugging:systematic process of relentlessly identifying the cause of a computer program that doesn't work.
  4. Technological empathy:understand what a computer is, how it works and what it's good and bad at doing.
  5. Systems thinking: break a big problem down into smaller pieces. Programmers do this when they create a plan (often on paper) for how a program will work. It involves big-picture thinking and decision-making about a problem and how different pieces of a program can work together to solve it.

Don't Repeat Yourself!

Another key aspect of coding is to avoid repetition or to copy/paste code. As this may lead to numerous errors. This is the so-called "Don't Repeat Yourself" principle in which each knowledge must have a single representation within a system. If this is applied correctly, a modification of any single element of a system does not require a change in other logically unrelated elements. Additionally, elements that are logically related all change predictably and uniformly
